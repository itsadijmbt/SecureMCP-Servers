

Changes
-------

1. Import swap (server.py).
   Removed (kept as comments above the new imports):
       from mcp.server.fastmcp import FastMCP, Image
       from mcp.shared.exceptions import McpError
       from mcp.types import ErrorData, INTERNAL_ERROR
   Added:
       from macaw_adapters.mcp import SecureMCP
       import base64

2. Class swap with composition wrapper (server.py).
   Was:  class OptunaMCP(FastMCP):
             super().__init__(...)
   Now:  class OptunaMCP:
             self.mcp = SecureMCP(name)
   Domain state (storage, study, dashboard) stays on the wrapper.
   Protocol concerns live on self.mcp.

3. Add tool() forwarder on the wrapper (server.py).
   def tool(self, *args, **kwargs):
       supported = {"name", "description", "prompts"}
       filtered = {k: v for k, v in kwargs.items() if k in supported}
       return self.mcp.tool(*args, **filtered)
   Why: ~15 sites use @mcp.tool(structured_output=True). SecureMCP
   only accepts (name, description, prompts). The forwarder filters
   the unsupported kwarg once instead of editing 15 call sites.
   Acceptable here because OptunaMCP has no public subclassers.

4. Forward run() (server.py).
   def run(self):
       self.mcp.run()

5. .model_dump() at BaseModel return sites (~11 places).
   Was:  return StudyResponse(study_name=name)
   Now:  return StudyResponse(study_name=name).model_dump()
   Why: SecureMCP serializes returns to JSON. pydantic BaseModel
   isn't JSON-native; .model_dump() converts to a plain dict at
   the boundary. Class definitions kept unchanged -- they still
   document the response shape.

6. Image returns rewritten as base64 dicts (~10 plot tools).
   Was:  return Image(data=plotly.io.to_image(fig), format="png")
   Now:  return {
             "image_format": "png",
             "image_base64": base64.b64encode(plotly.io.to_image(fig)).decode(),
         }
   Why: Image is FastMCP's protocol-aware binary type. SecureMCP
   doesn't ship it. The base64 dict is JSON-safe and any client
   can decode. Original `Image(...)` returns kept as `# was: ...`
   comments immediately above the new dicts.

7. McpError -> RuntimeError (~16 raise sites).
   Was:  raise McpError(ErrorData(code=INTERNAL_ERROR, message="..."))
   Now:  raise RuntimeError("...")
   Why: McpError is FastMCP's typed error envelope. SecureMCP
   catches exceptions generically; the specific class doesn't
   matter on the wire. Plain RuntimeError carries the same
   message with one fewer dependency. Originals kept as
   `# was: ...` comments above each replacement.

8. in *.toml pyversion >=3.11
