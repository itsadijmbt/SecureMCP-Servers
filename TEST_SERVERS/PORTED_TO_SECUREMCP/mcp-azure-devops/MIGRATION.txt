mcp-azure-devops: FastMCP -> SecureMCP port
============================================

This is the cleanest port shape we've seen so far. Two files
edited, four lines changed total. No tool function bodies
touched. No behaviour changes. No protocol-layer artifacts to
drop because the original didn't use any of them in the first
place (no annotations=, no icons=, no Field() defaults, no
ctx.<log> awaits, no _mcp_server access, no instructions=, no
Image returns, no lifespan).

-----------

NEW FILES
---------

client-test-macaw.py:
  Standard mesh smoke test. Two tests: tool-list sample
  (5 of 21 tools across distinct register paths) and
  get_projects (read-only Azure DevOps call; gracefully
  errors as a string when AZURE_DEVOPS_PAT is missing).
  Treats real-output-or-error-string as PASS for handler
  reachability.

WHY (one line each)
-------------------

1, 3. SecureMCP is the MACAW-instrumented MCP server class.
      Both modules need to import it so their references
      (constructor in server.py, type hint in
      conventions_promp.py) resolve at import time.
2. Constructor must reference the new class.
4. Python evaluates function annotations at definition time
   when from __future__ import annotations is not present.
   Leaving `mcp: FastMCP` after dropping the FastMCP import
   would raise NameError at module load.

BROKEN ON PURPOSE -- features intentionally lost
------------------------------------------------

None.

This port has no BROKEN ON PURPOSE entries because the
original mcp-azure-devops server did not use any of the
FastMCP-specific surfaces that SecureMCP doesn't carry:
  - No annotations= on any decorator -> nothing to drop.
  - No icons= on any decorator -> nothing to drop.
  - No instructions= constructor kwarg -> nothing to drop.
  - No mcp._mcp_server.* access -> Hard Rule #3 not triggered.
  - No Field(...) parameter defaults -> Step 2 not needed.
  - No await ctx.<log>() calls -> Step 5 not needed.
  - No FastMCP Image returns -> Step 2b not needed.
  - No @mcp.custom_route -> Step 6 not needed.
  - No lifespan -> Step 4 not needed.

The only thing the original used that does not survive
verbatim is the FastMCP class name itself, and that is what
the import swap addresses.

POSSIBLE ENVELOPE OBSERVATION (not a loss; just describing)
-----------------------------------------------------------

Most tools return strings (formatted Azure DevOps data or
"Error: ..." messages). SecureMCP wraps non-dict returns as
{"result": <str>} per mcp.py:815-817. Mesh consumers see the
same data inside a `result` key. Identical pattern documented
in spotify-mcp / telegram-mcp / k8s-mcp-server MIGRATION.txt
files; nothing new here.
