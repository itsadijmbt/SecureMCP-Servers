
Changes
-------

1. Import swap (mcp_server.py)
   - from mcp.server.fastmcp import FastMCP
   + from macaw_adapters.mcp import SecureMCP

2. Delete app_lifespan (mcp_server.py)
   The original @asynccontextmanager had no `await` in its body.
   It just built an AppContext dataclass and yielded it. We
   delete it entirely and move the AppContext init into
   utils.context as a module global, populated by
   initialize_app_context() called from main().

3. Drop FastMCP-only kwargs at construction (mcp_server.py)
   - mcp = FastMCP(MCP_SERVER_NAME, lifespan=app_lifespan, **config)
   + mcp = SecureMCP(MCP_SERVER_NAME)
   `lifespan=`, `host=`, `port=` are not accepted by SecureMCP.
   The CLI flags --host, --port, --transport stay on the surface
   for back-compat but are cosmetic; MACAW provides the mesh
   transport regardless.

4. Translate add_tool to decorator-as-function (mcp_server.py)
   - mcp.add_tool(tool, annotations=annotations)
   + mcp.tool(name=tool.__name__, description=tool.__doc__ or "")(tool)
   SecureMCP exposes only the decorator factory. `annotations=` is
   dropped because SecureMCP only accepts (name, description, prompts).

5. Add initialize_app_context() call (mcp_server.py)
   Right before `mcp.run()`:
       initialize_app_context(
           read_only_mode=read_only_mode,
           read_only_query_mode=read_only_query_mode,
       )
   Replaces what app_lifespan used to do. Must run before any tool
   call so get_app_context() returns a valid object.

6. Rewrite the choke-point helper (utils/context.py)
   Moved from "read AppContext through ctx.request_context" to two
   module-level globals:
       _APP_CONTEXT  -- the dataclass (read_only_mode etc.).
       _CLUSTER      -- the Couchbase cluster, lazy-built on first
                        tool call.
   Two new functions:
       initialize_app_context(read_only_mode, read_only_query_mode)
       get_app_context() -> AppContext
   And one rewritten:
       get_cluster_connection() -> Cluster   (no `ctx` arg anymore)
   The old `_set_cluster_in_lifespan_context(ctx)` helper is gone;
   its lazy-init body is inlined into get_cluster_connection().

7. Update the tool files (kv.py, query.py, server.py, index.py)
   Three mechanical edits per file:

   (a) Context import swap:
       - from mcp.server.fastmcp import Context
       + from macaw_adapters.mcp import Context

   (b) Drop the `ctx` arg from every cluster fetch:
       - cluster = get_cluster_connection(ctx)
       + cluster = get_cluster_connection()
       The original line is kept as a `# PORT:` comment above each
       new line for traceability.

   (c) Replace ctx.request_context.lifespan_context reads with
       module-global reads:
       - app_context = ctx.request_context.lifespan_context
       + app_context = get_app_context()        # query.py
       In server.py the cluster_connected check was rewritten to
       read utils.context._CLUSTER directly via the imported module
       (NOT via `from utils.context import _CLUSTER`, which would
       freeze a stale reference at import time).
