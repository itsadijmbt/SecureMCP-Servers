
File: src/mcp_server_qdrant/mcp_server.py

  a) Import swap
       before: from mcp.server.fastmcp import Context, FastMCP
       after : from macaw_adapters.mcp import Context, SecureMCP

     The Context type looks the same to the tool functions, but it now comes
     from MACAW and carries identity / audit / policy hooks internally.

  b) Base class swap
       before: class QdrantMCPServer(FastMCP):
       after : class QdrantMCPServer(SecureMCP):

     This single line is the "integration." By inheriting from SecureMCP,
     the server registers itself as a MACAW agent on startup and every
     tool it declares goes through MACAW's invoke_tool pipeline.

  c) __init__ rewrite
     - Added a `name` parameter with default "securemcp-server-qdrant".
     - Dropped the FastMCP-only `instructions` param (not used by SecureMCP).
     - Preserved all qdrant-specific args (tool_settings, qdrant_settings,
       embedding_provider_settings, embedding_provider) because the rest of
       qdrant's code still uses them.
     - Called super().__init__(name=name, **kwargs) exactly ONCE at the end,
       after building self.qdrant_connector. (Early drafts had a duplicate
       super() call + an undefined `**settings` expansion — that was a bug;
       we removed it.)

  d) Tool registration idiom
     FastMCP supports the dual-call shape:
         self.tool(fn, name="qdrant-find", description=...)
     SecureMCP only supports the decorator shape:
         self.tool(name="qdrant-find", description=...)(fn)

     We changed both registrations to the decorator-application form:

         self.tool(
             name="qdrant-find",
             description=self.tool_settings.tool_find_description,
         )(find_foo)

         if not self.qdrant_settings.read_only:
             self.tool(
                 name="qdrant-store",
                 description=self.tool_settings.tool_store_description,
             )(store_foo)

     IMPORTANT: we did not change SecureMCP to accept the dual-call shape.
     We adapted qdrant's call sites to match SecureMCP's idiom — that is
     the rule.

  e) Logging calls (ctx.debug)
     FastMCP's Context.debug is `async` (you had to `await` it).
     SecureMCP's Context.debug is SYNC.
     We removed the `await` in front of every ctx.debug call inside
     `store()` and `find()`. Leaving an `await` would raise
     "object NoneType can't be used in 'await' expression" at runtime.

  f) QdrantConnector stays a plain instance attribute
     The connector is process-scoped (one shared instance for all callers).
     We store it on `self.qdrant_connector` inside __init__. We do NOT put
     it in MACAW's _session_objects dict — that dict is per-session state,
     not process state. Qdrant doesn't have a per-user session model; one
     connector per process is correct.
